# Slug 기능 구현 계획서

프로젝트의 메뉴 상세 페이지 접근성을 개선하고 SEO(검색 엔진 최적화)를 강화하기 위한 **Slug** 기능 구현 계획입니다.

## 1. 개요
*   **목적**: 숫자로 된 ID(예: `/menus/12`) 대신 의미 있는 문자열(예: `/menus/iced-americano`)을 URL로 사용하여 사용자 경험과 검색 엔진 최적화를 향상시킵니다.
*   **현재 상태**: 
    *   데이터베이스 및 엔티티에 `slug` 필드 존재.
    *   조회 API(`GET /api/menus/slug/{slug}`) 구현됨.
    *   사용자 페이지(Frontend)에서 Slug 기반 라우팅 및 조회 로직 구현됨.
    *   **문제점**: 관리자 페이지에서 메뉴 생성/수정 시 `slug`를 입력할 수 없으며, 백엔드에서 필수값으로 설정되어 있어 오류 발생 가능성이 있음.

---

## 2. 세부 구현 계획

### 2.1 관리자 페이지 (Frontend)
*   **`MenuForm.tsx` 수정**:
    *   '영문 메뉴명' 아래에 **'슬러그(Slug)'** 입력 필드 추가.
    *   **자동 생성 로직**: 영문 메뉴명을 입력하면 자동으로 소문자 변환 및 하이픈(-) 처리를 거쳐 슬러그를 제안하는 기능 추가.
    *   **유효성 검사**: 영문, 숫자, 하이픈(-)만 허용하도록 Regex 검증 추가.

### 2.2 백엔드 (Java/Spring Boot)
*   **중복 체크**: 메뉴 저장 시 이미 존재하는 슬러그인지 확인하는 로직 서비스 레이어(`MenuService`)에 추가.
*   **자동 생성 (선택 사항)**: 만약 프론트엔드에서 슬러그가 전송되지 않을 경우를 대비하여 `engName`을 기반으로 `Menu` 도메인 모델 내부에서 슬러그를 생성하는 메서드 구현.
*   **데이터베이스 제약 조건**: `menus` 테이블의 `slug` 컬럼에 `UNIQUE` 인덱스 추가 (기존 데이터 검토 필요).

### 2.3 데이터 마이그레이션
*   기존 메뉴 데이터의 슬러그(Slug)가 비어있는 항목에 대해 `engName` 기반으로 값을 채워주는 SQL 스크립트 실행.

## 3. 신규 기능: AI 챗봇 장바구니 연동
메뉴 조회를 넘어 챗봇과의 대화를 통해 즉시 장바구니에 메뉴를 담는 기능을 추가합니다.

### 3.1 AI 에이전트 (Python/Gemini)
*   **Tool Calling 도입**: Gemini 모델에 아래 도구(Tool)들을 정의.
    *   `add_to_cart(slug: str, quantity: int)`: 메뉴를 장바구니에 담기.
    *   `navigate_to(path: str)`: 특정 페이지(메뉴 목록, 장바구니 등)로 이동하기.
*   **프롬프트 수정**: 사용자가 "메뉴 보여줘", "주문할래", "장바구니 담아줘"와 같이 요청할 때 각 도구를 적절히 호출하도록 유도.
*   **응답 스키마 확장**: 텍스트 응답뿐만 아니라 프론트엔드에서 해석할 수 있는 함수 호출 정보(Function Call)를 포함한 JSON 구조 반환.

### 3.2 프론트엔드 (Next.js)
*   **액션 처리 핸들러**: `ChatWidget`에서 AI의 응답에 포함된 도구 호출을 분석하여 실행.
    *   `navigate_to` 호출 시 Next.js `router.push()`를 사용하여 페이지 이동.
    *   `add_to_cart` 호출 시 `useCartStore`의 `addItem` 실행.
*   **피드백 강화**: 동작 수행 후 챗봇이 "메뉴 페이지로 모실게요! 🔥" 또는 "장바구니에 담았습니다! 🔥"와 같은 확인 메시지를 출력.

## 4. 기대 효과 및 확장성
1.  **공유하기 쉬운 URL**: 사용자가 메뉴 링크를 복사하여 공유할 때 어떤 메뉴인지 명확히 알 수 있음.
2.  **SEO 최적화**: 검색 엔진이 URL의 정보를 파악하여 관련 검색 결과에서 상위에 노출될 확률이 높아짐.
3.  **사이트 신뢰도**: 정적인 ID 기반보다 전문적인 웹 애플리케이션 느낌을 줌.
4.  **향후 확장성**: 메뉴뿐만 아니라 카테고리 등 다른 도메인에도 확장 적용 가능 및 역사적인 슬러그(Historical Slugs) 관리 고려.

## 5. 단계별 실행 일정
1. **1단계 (Back-office)**: 백엔드 Slug 자동 생성 로직 구현 및 관리자 페이지(`MenuForm`) Slug 입력/자동 제안 기능 추가. (완료 ✅)
2. **2단계 (AI Agent)**: Gemini Tool Calling 기능을 통한 `add_to_cart`, `navigate_to` 도구 정의 및 프롬프트 최적화. (완료 ✅)
3. **3단계 (Frontend)**: `ChatWidget`에서 AI의 도구 호출을 해석하여 페이지 이동 및 장바구니 액션을 수행하는 로직 연동. (완료 ✅)
4. **4단계 (Final)**: 전체 기능 통합 테스트 및 기존 데이터 마이그레이션 확인. (완료 ✅)
